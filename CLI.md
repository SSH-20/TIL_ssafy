# CLI 
CLI에 대해 알기 위하선 GUI에 대해 먼저 알아야한다.

CLI : Command Line Interface
-> 명령어를 통해 사용자와 컴퓨터가 상호작용하는 방식

그 이유는 WSL 환경에서 Linux에서 Docker를 사용할 것이기 때문이다.

왜 사용해야 할까?
- 효율성
  - 키보드만으로 모든 작업을 빠르고 가볍게 수행할 수 있음
- 정밀한 제어
  - 그래픽 화면에 보이지 않는 숨겨진 설정까지 정밀하게 제어 가능
- 강력한 자동화
  - 단순하고 반복적인 작업을 명령어 한 줄로 자동화 가능 : 명령어 한줄로 commit 등 가능
- 표준 환경
  - 서버, 클라우드 등 그래픽이 없는 개발 환경의 필수 도구


CLI에서 '.'(마침표)의 역할
- . : 현재 디렉토리 ex) ./
- .. : 현재의 상위 디렉토리 ex) ../

기초 문법
- touch : 파일 생성
- mkdir : 새 디렉토리 생성
- ls : 현재 작업 중인 디렉토리
- cd : 현재 작업 위치를 변경하는 명령어
* terminal 단축키 : ctrl + ` (백틱)
