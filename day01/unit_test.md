# TDD
> Test-Driven Develop  

코드를 작성할 때 한번에 모든 코드를 작성하여 합치고 테스트를 진행하지 말고 각 맡은 분야에서 테스트 코드를 작성하고 본 코드를 작성한다.  
-> 즉 유닛? 모듈? 단위로 테스트를 진행하면서 코드를 작성한다

- red, green, refector 세 가지로 나누어 코드를 작성한다.
- red : 테스트 코드를 작성하여 실패하는 케이스를 확인한다.
- green : 발생한 오류를 수정하는데 모든 수단을 동원한다. 하드 코딩이라도
- refector : 하드코딩 등의 방법을 제외한 방법들로 코드의 완성도를 높인다.
